GitHub 上有个叫 agency-agents 的项目，184 个 agent 人格档案，分成工程、设计、营销、销售等二十多个部门。它的一个卖点是"中文互联网生态专区"——小红书、公众号、抖音、知乎、B站、百度、微博、快手、私域、直播电商、淘系拼多多、小程序开发、飞书开发，都有单独档案。英文圈同类项目几乎没人做这块，看着新鲜。

我挑了其中 13 个档案实际读了一遍。结论是两层分化很明显。

真懂行的有 6 个：抖音、公众号、微信私域、直播电商、小程序、飞书。辨别标志很一致——有具体数字，有具体规则，有具体坑。抖音档案写"首 3 秒完成率优先于点赞"，这是只有自己投过千川的人才会强调的优先级。私域档案写"企微 4 次/月群发限额"，这是撞过墙的人才记得住的硬约束。小程序档案提分包加载和域名白名单，飞书档案的多维表格 API 示例能跑。这些档案拿出去给一线运营当起手 SOP，不丢人。

懂框架但缺细节的有 6 个：小红书、微博、B站、快手、百度 SEO、淘系电商。术语对，结构对，读起来顺，但经不起细抠。小红书没讲透薯条投放的节奏和冷启;B 站"花火"一笔带过;淘系讲了千人千面，但没讲直通车关键词卡位、拼多多砍价返利这种真正影响利润的机制;百度 SEO 还停在 ICP 备案和国内服务器这种十年前就写进教材的要求。像是读过行业报告但没操过盘。

最差的是知乎档案。还在强调 Column 专栏，对 2023 年之后官方号商业化的新机制只字不提。停在 2020 年的认知。

辨别是不是"真懂"不难。13 个档案的骨架几乎一模一样，Identity、Mission、Rules、Deliverables 四段式套下来,这是典型的 AI 批量生成模板。但真懂的档案会在模板里夹带私货——一个具体数字、一条平台内部规则、一句"这里会翻车"的提醒。B 级档案的"血肉"是从公开资料里抄来的，永远差这一口气。

这件事的启发是：AI 能把任何行业的框架生成得像模像样，但生成不出血泪细节。后者只有人操盘过才写得出来。所以看到一个庞大的、看起来很专业的 agent 库，默认态度不能是整包拿来用，而是挑。挑的办法就是找那些"只有操盘手才会说的话"。没有，就是外人凑出来的。

对这个项目本身，我的判断是：它比海外同类强一截（人家根本没这些平台的档案），但离"一线操盘手花名册"还差一截。6 个亮点 + 7 个 padding。用的时候心里有数。
